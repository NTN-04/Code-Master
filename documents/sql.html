Giới thiệu SQL

SQL (Structured Query Language) là một ngôn ngữ tiêu chuẩn để tương tác với các hệ quản trị cơ sở dữ liệu quan hệ (RDBMS). SQL được sử dụng để truy vấn, chèn, cập nhật và sửa đổi dữ liệu.

SQL có thể được sử dụng để thực hiện nhiều tác vụ, chẳng hạn như truy vấn dữ liệu từ cơ sở dữ liệu, chèn bản ghi mới, cập nhật bản ghi và tạo các bảng và cơ sở dữ liệu mới.

Cú pháp SQL Cơ bản

SQL bao gồm nhiều phần khác nhau:

  • DDL (Data Definition Language) - Sử dụng để định nghĩa cấu trúc cơ sở dữ liệu (CREATE, ALTER, DROP)
  • DML (Data Manipulation Language) - Sử dụng để thao tác dữ liệu (SELECT, INSERT, UPDATE, DELETE)
  • DCL (Data Control Language) - Sử dụng để điều khiển quyền truy cập (GRANT, REVOKE)
  • TCL (Transaction Control Language) - Sử dụng để quản lý giao dịch (COMMIT, ROLLBACK)

Câu lệnh SELECT

Câu lệnh SELECT được sử dụng để chọn dữ liệu từ một cơ sở dữ liệu. Dữ liệu được trả về được lưu trữ trong một bảng kết quả, được gọi là tập kết quả.

-- Chọn tất cả các cột
SELECT * FROM customers;

-- Chọn cột cụ thể
SELECT first_name, last_name FROM customers;

-- Chọn các cột khác nhau
SELECT DISTINCT country FROM customers;

-- Thêm bí danh cho cột
SELECT first_name AS fname, last_name AS lname
FROM customers;

Mệnh đề WHERE

Mệnh đề WHERE được sử dụng để lọc các bản ghi. Nó chỉ trích xuất những bản ghi đáp ứng điều kiện được chỉ định.

-- Toán tử so sánh
SELECT * FROM customers
WHERE country = 'USA';

-- Toán tử AND & OR
SELECT * FROM customers
WHERE country = 'USA' AND (age > 30 OR visits > 10);

-- Toán tử IN
SELECT * FROM customers
WHERE country IN ('USA', 'UK', 'Canada');

-- Toán tử BETWEEN
SELECT * FROM products
WHERE price BETWEEN 10 AND 50;

-- Toán tử LIKE
SELECT * FROM customers
WHERE last_name LIKE 'Sm%'; -- Tìm last_name bắt đầu bằng "Sm"

-- Toán tử IS NULL
SELECT * FROM customers
WHERE phone IS NULL;

JOIN

JOIN được sử dụng để kết hợp các hàng từ hai hoặc nhiều bảng, dựa trên một cột liên quan giữa chúng.

-- INNER JOIN (chỉ trả về các hàng khớp)
SELECT orders.order_id, customers.first_name, customers.last_name
FROM orders
INNER JOIN customers ON orders.customer_id = customers.id;

-- LEFT JOIN (trả về tất cả hàng từ bảng bên trái, và các hàng khớp từ bảng bên phải)
SELECT customers.first_name, orders.order_id
FROM customers
LEFT JOIN orders ON customers.id = orders.customer_id;

-- RIGHT JOIN (trả về tất cả hàng từ bảng bên phải, và các hàng khớp từ bảng bên trái)
SELECT customers.first_name, orders.order_id
FROM customers
RIGHT JOIN orders ON customers.id = orders.customer_id;

-- FULL JOIN (trả về tất cả hàng khi có khớp ở một trong hai bảng)
SELECT customers.first_name, orders.order_id
FROM customers
FULL JOIN orders ON customers.id = orders.customer_id;

GROUP BY

Mệnh đề GROUP BY nhóm các hàng có cùng giá trị vào các hàng tóm tắt, thường được sử dụng với các hàm tổng hợp như COUNT, MAX, MIN, SUM, AVG để nhóm tập kết quả theo một hoặc nhiều cột.

-- Đếm số khách hàng theo quốc gia
SELECT country, COUNT(*) AS total_customers
FROM customers
GROUP BY country;

-- Tính tổng số lượng theo danh mục sản phẩm
SELECT category, SUM(quantity) AS total_quantity
FROM products
GROUP BY category;

-- Tìm giá cao nhất trong mỗi danh mục sản phẩm
SELECT category, MAX(price) AS highest_price
FROM products
GROUP BY category;

HAVING

Mệnh đề HAVING được thêm vào SQL vì từ khóa WHERE không thể được sử dụng với các hàm tổng hợp. 

-- Chỉ hiển thị các quốc gia có nhiều hơn 5 khách hàng
SELECT country, COUNT(*) AS total_customers
FROM customers
GROUP BY country
HAVING COUNT(*) > 5;

-- Chỉ hiển thị danh mục sản phẩm có giá trung bình > $50
SELECT category, AVG(price) AS avg_price
FROM products
GROUP BY category
HAVING AVG(price) > 50;

ORDER BY

Mệnh đề ORDER BY được sử dụng để sắp xếp các bản ghi kết quả theo thứ tự tăng dần hoặc giảm dần. 

-- Sắp xếp khách hàng theo tên (tăng dần)
SELECT * FROM customers
ORDER BY first_name;

-- Sắp xếp khách hàng theo tên (giảm dần)
SELECT * FROM customers
ORDER BY first_name DESC;

-- Sắp xếp theo nhiều cột
SELECT * FROM customers
ORDER BY country, first_name;

INSERT, UPDATE, DELETE

Các câu lệnh INSERT, UPDATE và DELETE được sử dụng để sửa đổi dữ liệu trong bảng.

-- Chèn một bản ghi mới
INSERT INTO customers (first_name, last_name, email)
VALUES ('John', 'Doe', 'john.doe@example.com');

-- Chèn nhiều bản ghi
INSERT INTO customers (first_name, last_name, email)
VALUES 
('John', 'Doe', 'john.doe@example.com'),
('Jane', 'Smith', 'jane.smith@example.com');

-- Cập nhật bản ghi
UPDATE customers
SET email = 'new.email@example.com'
WHERE id = 1;

-- Xóa bản ghi
DELETE FROM customers
WHERE id = 1;

CREATE, ALTER, DROP

Các câu lệnh CREATE, ALTER và DROP được sử dụng để tạo, sửa đổi và xóa cấu trúc cơ sở dữ liệu. 

-- Tạo bảng
CREATE TABLE customers (
    id INT PRIMARY KEY AUTO_INCREMENT,
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    birth_date DATE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Thêm cột vào bảng
ALTER TABLE customers
ADD COLUMN phone VARCHAR(20);

-- Sửa đổi cột
ALTER TABLE customers
MODIFY COLUMN phone VARCHAR(30);

-- Xóa cột
ALTER TABLE customers
DROP COLUMN phone;

-- Xóa bảng
DROP TABLE customers;

-- Tạo cơ sở dữ liệu
CREATE DATABASE mydb;

-- Xóa cơ sở dữ liệu
DROP DATABASE mydb;

Ràng buộc

Các ràng buộc được sử dụng để chỉ định các quy tắc cho dữ liệu trong bảng.

-- Ràng buộc thường được sử dụng:
-- NOT NULL - Đảm bảo một cột không thể có giá trị NULL
-- UNIQUE - Đảm bảo tất cả các giá trị trong một cột là khác nhau
-- PRIMARY KEY - Kết hợp của NOT NULL và UNIQUE
-- FOREIGN KEY - Đảm bảo tính toàn vẹn tham chiếu
-- CHECK - Đảm bảo tất cả các giá trị trong một cột đáp ứng một điều kiện cụ thể
-- DEFAULT - Đặt một giá trị mặc định cho một cột khi không có giá trị nào được chỉ định

-- Ví dụ:
CREATE TABLE products (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    price DECIMAL(10, 2) CHECK (price > 0),
    category_id INT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (category_id) REFERENCES categories(id)
);
