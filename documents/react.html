Giới thiệu React

React là một thư viện JavaScript được phát triển bởi Facebook để xây dựng giao diện người dùng. Nó cho phép bạn tạo các thành phần giao diện người dùng có thể tái sử dụng và quản lý trạng thái của chúng một cách hiệu quả.

React sử dụng Virtual DOM để cải thiện hiệu suất bằng cách giảm thiểu các thao tác DOM thật. Điều này làm cho React rất phù hợp cho các ứng dụng một trang (SPAs) nơi cần phải cập nhật UI thường xuyên mà không làm mới toàn bộ trang.

Components

Components là các khối xây dựng của bất kỳ ứng dụng React nào. Một component là một phần độc lập, có thể tái sử dụng của giao diện người dùng.

Có hai loại components trong React:

  • Function Components - Các hàm JavaScript đơn giản trả về các phần tử React
  • Class Components - Các lớp ES6 mở rộng từ React.Component 

  // Function Component
  function Welcome(props) {
    return <h1>Hello, {props.name}</h1>;
  }

  // Class Component
  class Welcome extends React.Component {
    render() {
      return <h1> Hello, {this.props.name}</h1>;
    }
  }

JSX

JSX là một cú pháp mở rộng cho JavaScript được đề xuất bởi React. Nó trông giống như HTML hoặc XML trong code JavaScript của bạn.

JSX không bắt buộc phải sử dụng với React, nhưng nó làm cho code React dễ viết và dễ đọc hơn. 


    // JSX
  const element = <h1>Hello, world!</h1>;

  // JSX với biểu thức JavaScript
  const name = 'John';
  const element = <h1>Hello, {name}!</h1>;

  // JSX với nhiều phần tử (cần một phần tử gốc)
  const element = (
    <div>
      <h1>Hello!</h1>
      <p>Good to see you here.</p>
    </div>
  );

Props

Props (viết tắt của "properties") là cách components trong React giao tiếp với nhau. Props là đối số được truyền cho các components, tương tự như các thuộc tính HTML.

Props là chỉ đọc và không thể được sửa đổi từ bên trong component nhận chúng. 

// Truyền props
  <Welcome name="Sara" />

  // Nhận props trong function component
  function Welcome(props) {
    return <h1>Hello, {props.name}</h1>;
  }

  // Nhận props trong class component
  class Welcome extends React.Component {
    render() {
      return <h1>Hello, {this.props.name}</h1>;
    }
  }

State

State là dữ liệu riêng tư của component. State có thể thay đổi theo thời gian và ảnh hưởng đến cách component được hiển thị.

Khác với props, state có thể được sửa đổi bởi component sở hữu nó. 


 // State trong class component
  class Counter extends React.Component {
    constructor(props) {
      super(props);
      this.state = { count: 0 };
    }
    
    increment = () => {
      this.setState({ count: this.state.count + 1 });
    }
    
    render() {
      return (
        <div>
          <p>Count: {this.state.count}</p>
          <button onClick={this.increment}>Increment</button>
        </div>
      );
    }
  }

  // State với useState hook trong function component
  function Counter() {
      const [count, setCount] = useState(0);
      return (
        <div>
          <p>Count: {count}</p>
          <button onClick= {() => setCount(count + 1)}>
            Increment
          </button>
        </div>
      );
    }

Hooks

Hooks là một tính năng được giới thiệu trong React 16.8 cho phép bạn sử dụng state và các tính năng React khác mà không cần viết một class.

Hooks phổ biến nhất bao gồm:

  • useState - Cho phép thêm state vào function components
  • useEffect - Quản lý side effects trong function components
  • useContext - Truy cập context từ function components
  • useReducer - Quản lý state phức tạp và các cập nhật liên quan đến state trước đó
  • useRef - Tham chiếu đến các phần tử DOM hoặc lưu trữ giá trị không gây re-render 

  // useState
  function Counter() {
    const [count, setCount] = useState(0);
    return (
      <div>
        <p>Count: {count}</p>
        <button onClick={() => setCount(count + 1)}>Increment</button>
      </div>
    );
  }

      // useEffect
    function Example() {
    const [count, setCount] = useState(0);
    
    useEffect(() => {
      document.title = `You clicked ${count} times`;
    }, [count]); // Chỉ chạy lại khi count thay đổi
    
    return (
      <div>
        <p>You clicked {count} times</p>
        <button onClick={() => setCount(count + 1)}>
          Click me
        </button>
      </div>
    );
  }

Side Effects

Side effects là các hoạt động không phải là một phần của chu trình render, chẳng hạn như yêu cầu dữ liệu, thiết lập đăng ký hoặc thay đổi DOM thủ công.

Trong class components, side effects thường được thực hiện trong các phương thức lifecycle như componentDidMount, componentDidUpdate và componentWillUnmount. Trong function components, useEffect hook được sử dụng để xử lý side effects. 

// Fetch data với useEffect
  function UserProfile({ userId }) {
    const [user, setUser] = useState(null);
    const [loading, setLoading] = useState(true);
    const [error, setError] = useState(null);
    
    useEffect(() => {
      async function fetchUser() {
        try {
          setLoading(true);
          const response = await fetch(`https://api.example.com/users/${userId}`);
          const data = await response.json();
          setUser(data);
          setError(null);
        } catch (err) {
          setError('Failed to fetch user data');
          setUser(null);
        } finally {
          setLoading(false);
        }
      }
      
      fetchUser();
    }, [userId]); // Chỉ chạy lại khi userId thay đổi
    
    if (loading) return <p>Loading...</p>;
    if (error) return <p>{error}</p>;
    if (!user) return null;
    
    return (
      <div>
        <h1>{user.name}</h1>
        <p>Email: {user.email}</p>
      </div>
    );
  }

React Router

React Router là một thư viện định tuyến tiêu chuẩn cho React. Nó cho phép bạn xử lý định tuyến trong ứng dụng React, cho phép bạn tạo một ứng dụng một trang (SPA) với điều hướng mà không cần làm mới trang. 

// Cài đặt
  // npm install react-router-dom

  // Sử dụng cơ bản
  import { BrowserRouter, Routes, Route, Link } from 'react-router-dom';

  function App() {
    return (
      <BrowserRouter>
        <nav>
          <Link to="/">Home</Link>
          <Link to="/about">About</Link>
        </nav>
        
        <Routes>
          <Route path="/" element={ < Home />} />
          <Route path="/about" element={ < About />} />
          <Route path="*" element={ < NotFound /> } />
        </Routes>
      </BrowserRouter>
    );
  }

Quản lý State

Khi ứng dụng React của bạn phát triển, quản lý state có thể trở nên phức tạp. Có một số giải pháp quản lý state phổ biến:

  • Context API - Được tích hợp sẵn trong React, giúp truyền dữ liệu qua cây component mà không cần truyền props qua nhiều cấp
  • Redux - Một thư viện quản lý state có thể dự đoán cho các ứng dụng JavaScript
  • MobX - Một giải pháp quản lý state đơn giản, có thể mở rộng và mạnh mẽ
  • Recoil - Một thư viện quản lý state do Facebook phát triển đặc biệt cho React 
// Context API
  const ThemeContext = React.createContext('light');

  function App() {
    return (
      <ThemeContext.Provider value="dark">
        <Toolbar />
      </ThemeContext.Provider>
    );
  }

  function Toolbar() {
    return <ThemedButton />;
  }

  function ThemedButton() {
    const theme = useContext(ThemeContext);
    return <button className={theme}>Themed Button</button>;
  }
