Giới thiệu CSS

CSS (Cascading Style Sheets) là ngôn ngữ được sử dụng để tạo kiểu cho trang web HTML. CSS mô tả cách các phần tử HTML nên được hiển thị trên màn hình, trên giấy, hoặc trong các phương tiện khác.

CSS tiết kiệm rất nhiều công việc. Nó có thể kiểm soát bố cục của nhiều trang web cùng lúc.

Cú pháp CSS cơ bản:

Selector { property:value; }

Trong đó:

  • Selector: Các đối tượng mà chúng ta sẽ áp dụng các thuộc tính trình bày. Nó là các tag HTML, class hay id (chúng ta sẽ học về 2 phần này ở bài học sau). Ví dụ: body, h2, p, img, #title, #content, .username,...

Trong CSS ngoài viết tên selector theo tên tag, class, id. Chúng ta còn có thể viết tên selector theo phân cấp như để chỉ các ảnh ở trong #entry, chúng ta viết selector là #entry img, như vậy thì các thuộc tính chỉ định sẽ chỉ áp dụng riêng cho các ảnh nằm trong #entry.

Khi viết tên cho class, đôi khi sẽ có nhiều thành phần có cùng class đó, ví dụ như thẻ img và thẻ a cùng có class tên vistors nhưng đây lại là hai đối tượng khác nhau, 1 cái là ảnh của người thăm, 1 cái là liên kết tới trang người thăm. Nên nếu khi viết CSS ta ghi là .visitors { width:50 } thì sẽ ảnh hưởng tới cả hai thành phần. Nên trong trường hợp này, nếu bạn có ý dùng CSS đó chỉ riêng phần ảnh thì chỉ nên ghi là img .visitors.

Một lối viết selector nữa là dựa trên tên các thuộc tính có trong HTML. Ví dụ trong HTML ta có đoạn mã như vầy: <input name="Search" type="Text" value="Key Word">. Để áp dụng thuộc tính CSS cho riêng ô tìm kiếm này chúng ta sẽ dùng selector input[name="Search"].

Ngoài việc viết tên selector cụ thể, chúng ta cũng có thể dùng một selector đại diện như * { color:red } sẽ tác động đến tất cả các thành phần có trên trang web làm cho chúng có text màu đỏ.

  • Property: Chính là các thuộc tính quy định cách trình bày. Ví dụ: background-color, font-family, color, padding, margin,...

Mỗi thuộc tính CSS phải được gán một giá trị. Nếu có nhiều hơn một thuộc tính cho một selector thì chúng ta phải dùng một dấu chấm phẩy (;) để phân cách các thuộc tính. Tất cả các thuộc tính trong một selector sẽ được đặt trong một cặp ngoặc nhọn sau selector.

Ví dụ:

body { background:#FFF; color:#FF0000; font-size:14pt }

Để dễ đọc hơn, bạn nên viết mỗi thuộc tính CSS ở một dòng. Tuy nhiên, nó sẽ làm tăng dung lượng lưu trữ CSS của bạn.

Ví dụ:


  body {

    background:#FFF;

    bolor:#FF0000;

    font-size:14pt

}
        
/* Đây là một bình luận CSS */
body {
  background-color: lightblue;
}

h1 {
  color: navy;
  margin-left: 20px;
}

Bộ chọn CSS

Bộ chọn CSS được sử dụng để "tìm" (hoặc chọn) các phần tử HTML bạn muốn tạo kiểu.

Chúng ta có thể chia các bộ chọn CSS thành:

Bộ chọn ID:

Bộ chọn ID là một cách để chúng ta có thể chọn một phần tử mà không cần chọn phần tử khác cùng loại. Giả sử chúng ta cập nhật mã HTML trong <article> ví dụ này để bài viết đầu tiên được "gắn thẻ" bằng một ID: 


  <article id="featured">
    <!-- Article 1 -->
  </article>
  
  <article>
    <!-- Article 2 -->
  </article>
  
  <article>
    <!-- Article 3 -->
  </article>

Bây giờ chúng ta có thể sử dụng ID đó để phân biệt bài viết đầu tiên với các bài viết khác và áp dụng các kiểu cụ thể cho bài viết đó. Chúng ta thêm ký tự hashtag ( #) vào trước tên ID khi viết bộ chọn CSS để chọn đúng ID. 


  /* Selects all 
 elements */
article {
  background: #fff;
}

/* Selects any element with id="featured" */
#featured {
  background: hsl(35 100% 90%);
  border-color: hsl(35 100% 50%);
}

Bộ chọn Class:

Bộ chọn lớp có thể là loại bộ chọn CSS được sử dụng phổ biến nhất mà bạn sẽ thấy trên web. Lớp là lựa chọn lý tưởng vì chúng cụ thể hơn một chút so với bộ chọn phần tử nhưng không quá phức tạp như ID.

  • Bộ chọn CSS .classchọn các phần tử có giá trị thuộc tính lớp cụ thể.
  • Để chọn tất cả các loại phần tử có lớp cụ thể, hãy viết ký tự dấu chấm (.), theo sau là giá trị thuộc tính lớp.
  • Bộ chọn sẽ chọn các phần tử được chỉ định với giá trị thuộc tính lớp được chỉ định. element.class

Đó là lý do khiến bộ chọn lớp trở nên phổ biến — chúng chỉ cụ thể hơn một chút so với các phần tử, nhưng vẫn giữ được độ cụ thể ở mức đủ thấp để có thể quản lý được nếu chúng ta cần ghi đè các kiểu trong một bộ quy tắc bằng các kiểu trong một bộ quy tắc khác.

Sự khác biệt duy nhất khi viết một lớp là chúng ta thêm dấu chấm ( .) vào trước tên lớp thay vì dấu thăng ( #).


 .intro {
  background-color: yellow;
}

Bộ chọn phần tử:

Bộ chọn CSS sẽ chọn tất cả các phần tử có tên phần tử được chỉ định. 


 h1 {
  border: 2px solid green;
  background-color: beige;
}

p {
  background-color: yellow;
}

Bộ chọn kết hợp:

Nếu bộ chọn là "cái gì" chúng ta chọn trong CSS, thì bạn có thể nghĩ về bộ kết hợp CSS là "cách" chúng ta chọn chúng. Chúng được sử dụng để viết các bộ chọn kết hợp các bộ chọn khác nhằm mục tiêu đến các phần tử. Khởi đầu!

Cái tên "combinator" thật tuyệt vời vì nó truyền tải chính xác nhiều cách khác nhau mà chúng ta có thể kết hợp các bộ chọn . Tại sao chúng ta cần kết hợp các bộ chọn? Như đã thảo luận trước đó về Bộ chọn Chuỗi, có hai trường hợp phổ biến mà chúng ta cần thực hiện điều này:

  • Khi chúng ta muốn tăng tính cụ thể của nội dung được chọn.
  • Khi chúng ta muốn chọn một phần tử dựa trên một điều kiện.

Một bộ kết hợp con thực chất chỉ là một nhánh của bộ kết hợp con, chỉ khác là nó cụ thể hơn bộ kết hợp con vì nó chỉ chọn các phần tử con trực tiếp của một phần tử , chứ không phải bất kỳ phần tử con nào.


 <div class="parent">
  <div class="child"></div>
  <div class="child">
    <div class="grandchild"></div>
  </div>
  <div class="child"></div>
  <div class="child"></div>
</div>

Mô hình hộp

Trong CSS, mô hình hộp là một hộp bao quanh mọi phần tử HTML. Nó bao gồm: lề (margins), đường viền (borders), phần đệm (padding) và nội dung thực tế.

Mô hình hộp cho phép chúng ta thêm đường viền xung quanh phần tử và xác định không gian xung quanh phần tử. 

div {
  width: 300px;
  padding: 25px;
  border: 5px solid navy;
  margin: 25px;
}

Thuộc tính Margin

thuộc tính margin trong CSS cũng được dùng để canh lề cho cả trang web hay một thành phần web này với các thành phần web khác hay với viền trang

Đặt lề cho cả bốn cạnh của phần tử:

p {
  margin: 35px;
}

Nếu thuộc tính lề có ba giá trị:

p {
  margin: 35px 30px 20px; // trên - trái, phải - dưới
}

Nếu thuộc tính lề có hai giá trị:

p {
  margin: 35px 30px; // trên, dưới - trái, phải  
}

Thuộc tính Padding

Padding có thể hiểu như là một thuộc tính đệm. Padding không ảnh hưởng tới khoảng các giữa các các đối tượng như margin mà nó chỉ quy định khoảng cách giữa phần nội dung và viền của một đối tượng

Cú pháp: Tương tự margin.

p {
  padding: 35px 30px 20px 10px;
}

Bố cục CSS

CSS cung cấp nhiều thuộc tính khác nhau để kiểm soát cách các phần tử được bố trí trong trang web:

  • display - Xác định loại hộp hiển thị của phần tử
  • position - Xác định phương pháp định vị được sử dụng cho phần tử
  • float - Xác định cách phần tử nên float
  • clear - Xác định bên nào của phần tử không được phép nổi

Display: thuộc tính này display chỉ định hành vi hiển thị (loại hộp hiển thị) của một phần tử.


p.ex1 {display: none;}
p.ex2 {display: inline;}
p.ex3 {display: block;}
p.ex4 {display: inline-block;}

+ inline: Hiển thị một phần tử dưới dạng phần tử nội tuyến (như <span>). Bất kỳ thuộc tính chiều cao và chiều rộng nào cũng sẽ không có hiệu lực. Đây là mặc định.

+ block: Hiển thị một phần tử dưới dạng một phần tử khối (như <p>). Nó bắt đầu trên một dòng mới và chiếm toàn bộ chiều rộng.

+ inline-block: Hiển thị một phần tử dưới dạng một khối chứa nội tuyến. Bản thân phần tử được định dạng là một phần tử nội tuyến, nhưng bạn có thể áp dụng các giá trị chiều cao và chiều rộng.

+ none: Phần tử đã bị loại bỏ hoàn toàn

Float: thuộc tính này float chỉ định liệu một phần tử có nên trôi sang trái, sang phải hay không trôi.

Lưu ý: Các thành phần được định vị tuyệt đối sẽ bỏ qua float thuộc tính này!

Lưu ý: Các phần tử bên cạnh một phần tử nổi sẽ chảy xung quanh nó. Để tránh điều này, hãy sử dụng thuộc tính clear

Cú pháp: float: none | left | right | initial | inherit;


img  {
  float: right; // nổi về bên phải
}

+ right: Cố định phần tử về bên phải

+ left: Cố định phần tử về bên trái

+ none: bình thường

Clear: Đi cùng với thuộc tính float, trong CSS còn có một thuộc tính là clear. Thuộc tính clear là một thuộc tính thường được gán vào các phần tử liên quan tới phần tử đã được float để quyết định hướng xử sự của phần tử này.

Phần tử <p> được đẩy xuống bên dưới các phần tử nổi bên trái (phần tử <p> không cho phép các phần tử nổi ở bên trái):


img {
  float: left;
}

p.clear {
  clear: left;
}

Position:

Kết hợp với thuộc tính float đã học, thuộc tính position mang lại nhiều khả năng để tạo một cách trình bày tiên tiến và chính xác cho trang web.

Hãy tưởng tượng cửa sổ trình duyệt của bạn giống như một hệ tọa độ và với positionbạn có thể đặt một đối tượng web ở bất cứ vị trí nào trên hệ tọa độ này.


// Cú pháp
position: static;
position: relative;
position: absolute;
position: fixed;
position: sticky;

static : Phần tử được định vị theo Luồng chuẩn của tài liệu. Các thuộc tính top, right, bottom, left, và không có hiệu lực . Đây là giá trị mặc định.z-index

absolute: Phần tử được xóa khỏi luồng tài liệu thông thường và không có khoảng trống nào được tạo cho phần tử đó trong bố cục trang. Phần tử được định vị so với phần tử tổ tiên gần nhất (nếu có) hoặc so với khối chứa ban đầu . Vị trí cuối cùng của nó được xác định bởi các giá trị top, right, bottom, và left.

relative: Phần tử được định vị theo luồng thông thường của tài liệu, sau đó được bù trừ so với chính nó dựa trên các giá trị top, right, bottom, và left. Việc bù trừ không ảnh hưởng đến vị trí của bất kỳ phần tử nào khác; do đó, không gian được cung cấp cho phần tử trong bố cục trang giống như khi vị trí là static.

fixed: Phần tử được xóa khỏi luồng tài liệu thông thường và không có khoảng trống nào được tạo cho phần tử đó trong bố cục trang. Phần tử được định vị so với khối chứa ban đầu của nó , tức là khung nhìn trong trường hợp phương tiện trực quan. Vị trí cuối cùng của nó được xác định bởi các giá trị top, right, bottom, và left.

Flexbox

Flexbox (Flexible Box Layout) là một mô hình bố cục trong CSS giúp sắp xếp, căn chỉnh và phân phối các phần tử con trong một container một cách linh hoạt – đặc biệt hữu ích khi xây dựng giao diện responsive (tự điều chỉnh theo kích thước màn hình).

Flexbox hoạt động dựa trên hai khái niệm chính:
- Container cha (thường gọi là "flex container") áp dụng display: flex;;
- Các phần tử con trong container sẽ trở thành flex items

Các thuộc tính cơ bản của Flexbox

1. Thuộc tính trên container cha

Thuộc tính Chức năng Giá trị tiêu biểu
display Kích hoạt flexbox layout flex hoặc inline-flex
flex-direction Hướng của trục chính row, column, row-reverse
justify-content Căn chỉnh phần tử theo trục chính flex-start, center, space-between, space-around
align-items Căn chỉnh phần tử theo trục phụ stretch, center, flex-start, flex-end
flex-wrap Cho phép phần tử xuống dòng khi tràn nowrap, wrap, wrap-reverse
gap Khoảng cách giữa các phần tử VD: gap: 10px;

2. Thuộc tính trên phần tử con

Thuộc tính Chức năng
flex-grow Mở rộng phần tử để lấp đầy không gian
flex-shrink Thu nhỏ phần tử khi không đủ không gian
flex-basis Kích thước ban đầu trước khi co giãn
flex Viết tắt của 3 thuộc tính trên (grow shrink basis)
align-self Căn chỉnh riêng phần tử con
order Thay đổi thứ tự hiển thị của phần tử

3. Ví dụ minh họa


// HTML
<div class="container">
  <div class="item">1</div>
  <div class="item">2</div>
  <div class="item">3</div>
</div>
              

// CSS
.container {
  display: flex;
  flex-direction: row;
  justify-content: space-around;
  align-items: center;
  height: 200px;
  background-color: #f0f0f0;
}

.item {
  width: 50px;
  height: 50px;
  background-color: #4caf50;
  color: white;
  text-align: center;
  line-height: 50px;
}
Kết quả: Ba ô vuông sẽ được căn giữa theo chiều dọc, phân bố đều theo chiều ngang với khoảng cách như nhau.

CSS Grid

CSS Grid Layout là hệ thống bố cục dạng lưới (grid) giúp chia nội dung thành hàng và cột. Grid đặc biệt hữu ích khi xây dựng bố cục hai chiều (dọc và ngang), tạo nên giao diện web phức tạp một cách dễ dàng.
CSS Grid có thể coi là “bảng vẽ” giúp bạn sắp xếp các phần tử một cách trực quan, linh hoạt, và hiệu quả.

Các thành phần chính trong Grid

1. Grid Container

Bạn kích hoạt bằng cách thêm display: grid; hoặc display: inline-grid; vào phần tử cha.

Thuộc tính Chức năng
grid-template-columns Định nghĩa số lượng và kích thước các cột
grid-template-rows Định nghĩa số lượng và kích thước các hàng
gap hoặc grid-gap Khoảng cách giữa các hàng và cột
justify-content Căn chỉnh nội dung theo chiều ngang
align-content Căn chỉnh nội dung theo chiều dọc

2. Grid Item

Các phần tử con nằm trong container grid sẽ có thể được đặt vào vị trí tùy ý.

Thuộc tính Chức năng
grid-column Chiếm bao nhiêu cột
grid-row Chiếm bao nhiêu hàng
grid-area Gộp khu vực gồm cả hàng và cột
justify-self Căn chỉnh phần tử theo chiều ngang
align-self Căn chỉnh phần tử theo chiều dọc

Ví dụ minh họa


<div class="grid-container">
  <div class="item item1">1</div>
  <div class="item item2">2</div>
  <div class="item item3">3</div>
</div>
    

.grid-container {
  display: grid;
  grid-template-columns: 100px 1fr 100px;
  grid-template-rows: 100px auto;
  gap: 10px;
  background-color: #eee;
}

.item {
  background-color: #2196f3;
  color: white;
  text-align: center;
  padding: 20px;
}

.item1 {
  grid-column: 1 / 3;
}

.item2 {
  grid-row: 2 / 3;
  grid-column: 2 / 4;
}

Thiết kế đáp ứng (Responsive Web Design)

Responsive là phương pháp thiết kế giao diện website sao cho nội dung hiển thị phù hợp với nhiều kích thước màn hình khác nhau — từ điện thoại, máy tính bảng đến máy tính để bàn. Thay vì tạo nhiều phiên bản giao diện, responsive cho phép một giao diện tự điều chỉnh linh hoạt.

Các kỹ thuật chính trong thiết kế responsive

1. Media Queries

Media Queries là công cụ chính trong CSS để áp dụng style khác nhau cho từng loại thiết bị.
Ví dụ: 


/* Áp dụng cho màn hình nhỏ hơn 600px */
@media (max-width: 600px) {
  .container {
    flex-direction: column;
  }
}

Từ @media là mấu chốt để kiểm tra điều kiện màn hình và thay đổi cách hiển thị giao diện.

2. Grid và Flexbox

Kết hợp Flexbox và Grid để tạo bố cục linh hoạt:
- Dùng flex-wrap: wrap; để các phần tử tự xuống hàng
- Dùng grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)); để grid tự co giãn tùy theo kích thước màn hình 


.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(150px, 1fr));
  gap: 20px;
}

3. Đơn vị đo linh hoạt

Thay vì px, dùng các đơn vị như:

Đơn vị Ý nghĩa
% Tính theo phần trăm của phần tử cha
vw/vh Theo kích thước cửa sổ trình duyệt
em/rem Tính theo kích thước font chữ

4. Responsive Image

Sử dụng ảnh linh hoạt để hiển thị phù hợp: 


<img src="large.jpg" alt="Ảnh lớn" 
  srcset="small.jpg 600w, medium.jpg 1000w"
  sizes="(max-width: 600px) 100vw, 50vw">

Trình duyệt sẽ chọn ảnh phù hợp theo độ rộng màn hình.
