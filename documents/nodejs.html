Giới thiệu Node.js

Node.js là một môi trường thời gian chạy JavaScript được xây dựng trên Chrome's V8 JavaScript engine. Node.js sử dụng một mô hình I/O không chặn, hướng sự kiện, khiến nó nhẹ và hiệu quả, lý tưởng cho các ứng dụng thời gian thực chạy trên các thiết bị phân tán.

Node.js cho phép bạn chạy JavaScript trên máy chủ, cho phép phát triển ứng dụng fullstack với cùng một ngôn ngữ lập trình.

Modules

Node.js sử dụng một hệ thống module nhẹ được gọi là "CommonJS". Modules cho phép bạn tổ chức code thành các phần nhỏ, có thể quản lý được.

Có ba loại module trong Node.js:

  • Core Modules - Được tích hợp sẵn trong Node.js
  • Local Modules - Được tạo bởi chính bạn
  • Third Party Modules - Được cài đặt thông qua npm
// Nhập core module
const fs = require('fs');

// Nhập local module
const myModule = require('./my-module');

// Nhập third party module
const express = require('express');

// Tạo module
// file: my-module.js
const myFunction = () => {
  console.log('Hello from my module');
};

module.exports = {
  myFunction
};

Hệ thống file

Module fs (file system) trong Node.js cung cấp một API để tương tác với hệ thống tệp theo một cách tương tự như các hàm POSIX tiêu chuẩn.

Tất cả các phương thức trong module fs có các phiên bản đồng bộ và bất đồng bộ.

const fs = require('fs');

// Đọc file (bất đồng bộ)
fs.readFile('file.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});

// Đọc file (đồng bộ)
try {
  const data = fs.readFileSync('file.txt', 'utf8');
  console.log(data);
} catch (err) {
  console.error(err);
}

// Ghi file
fs.writeFile('file.txt', 'Hello World!', (err) => {
  if (err) throw err;
  console.log('File đã được lưu!');
});

// Tạo thư mục
fs.mkdir('my-directory', (err) => {
  if (err) throw err;
  console.log('Thư mục đã được tạo!');
});

HTTP Server

Node.js có một module HTTP tích hợp sẵn cho phép bạn tạo các máy chủ web và khách hàng HTTP.

Máy chủ HTTP có thể lắng nghe các cổng máy chủ và trả về các phản hồi HTTP cho khách hàng.

const http = require('http');

// Tạo máy chủ HTTP
const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello, World!\n');
});

// Server lắng nghe trên cổng 3000
server.listen(3000, 'localhost', () => {
  console.log('Server đang chạy tại http://localhost:3000/');
});

Express.js

Express là một framework web tối thiểu và linh hoạt cho Node.js. Nó cung cấp một bộ tính năng mạnh mẽ để phát triển các ứng dụng web và di động.

Express giúp quản lý tất cả mọi thứ từ các routes trên máy chủ của bạn đến xử lý các yêu cầu và views.

const express = require('express');
const app = express();
const port = 3000;

// Middleware cho phép xử lý dữ liệu JSON
app.use(express.json());

// Tạo route đơn giản
app.get('/', (req, res) => {
  res.send('Hello, World!');
});

// Route với path parameter
app.get('/users/:userId', (req, res) => {
  res.send(`User ID: ${req.params.userId}`);
});

// POST route
app.post('/users', (req, res) => {
  // req.body chứa dữ liệu được gửi
  res.status(201).json({ 
    message: 'User created', 
    user: req.body 
  });
});

// Bắt đầu server
app.listen(port, () => {
  console.log(`Server đang chạy tại http://localhost:${port}`);
});

NPM

NPM (Node Package Manager) là một trình quản lý gói mặc định cho Node.js. Nó bao gồm một kho lưu trữ trực tuyến cho các gói Node.js và một dòng lệnh để cài đặt các gói đó.

NPM cho phép bạn dễ dàng quản lý các phụ thuộc trong dự án của bạn và cài đặt các gói của bên thứ ba.

// Khởi tạo dự án mới (tạo package.json)
npm init

// Cài đặt gói
npm install express

// Cài đặt gói như phụ thuộc phát triển
npm install nodemon --save-dev

// Cài đặt gói toàn cầu
npm install -g nodemon

// Xem danh sách gói đã cài đặt
npm list

// Xóa gói
npm uninstall express

// Chạy script đã định nghĩa trong package.json
npm run start

Lập trình bất đồng bộ

Node.js được xây dựng với sự bất đồng bộ trong tâm trí. Hầu hết các API Core của Node.js đều không chặn (non-blocking), có nghĩa là Node.js tiếp tục xử lý trong khi đợi callback.

Node.js hỗ trợ lập trình bất đồng bộ thông qua callbacks, promises và async/await.

// Callbacks
fs.readFile('file.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});

// Promises
const readFilePromise = (path) => {
  return new Promise((resolve, reject) => {
    fs.readFile(path, 'utf8', (err, data) => {
      if (err) reject(err);
      else resolve(data);
    });
  });
};

readFilePromise('file.txt')
  .then(data => console.log(data))
  .catch(err => console.error(err));

// Async/Await
const { promises: fsPromises } = require('fs');

async function readFile() {
  try {
    const data = await fsPromises.readFile('file.txt', 'utf8');
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}

readFile();

Kết nối cơ sở dữ liệu

Node.js có thể kết nối với nhiều loại cơ sở dữ liệu khác nhau thông qua các module bên thứ ba.

Các cơ sở dữ liệu phổ biến được sử dụng với Node.js bao gồm MongoDB, MySQL, PostgreSQL, Redis và nhiều loại khác.

// MongoDB với Mongoose
const mongoose = require('mongoose');

// Kết nối đến MongoDB
mongoose.connect('mongodb://localhost:27017/myapp', {
  useNewUrlParser: true,
  useUnifiedTopology: true
});

// Định nghĩa Schema
const userSchema = new mongoose.Schema({
  name: String,
  email: String,
  age: Number
});

// Tạo Model
const User = mongoose.model('User', userSchema);

// Tạo và lưu user mới
const newUser = new User({
  name: 'John Doe',
  email: 'john@example.com',
  age: 30
});

newUser.save()
  .then(user => console.log('User saved:', user))
  .catch(err => console.error('Error saving user:', err));

// MySQL với node-mysql
const mysql = require('mysql');

// Tạo kết nối
const connection = mysql.createConnection({
  host: 'localhost',
  user: 'user',
  password: 'password',
  database: 'mydb'
});

// Kết nối đến database
connection.connect();

// Thực thi query
connection.query('SELECT * FROM users', (error, results) => {
  if (error) throw error;
  console.log('Users:', results);
});

// Đóng kết nối
connection.end();

Bảo mật

Bảo mật là một khía cạnh quan trọng của bất kỳ ứng dụng Node.js nào. Dưới đây là một số phương pháp hay nhất để bảo mật ứng dụng Node.js của bạn:

  • Sử dụng HTTPS - Mã hóa dữ liệu được truyền giữa máy chủ và máy khách
  • Validate Input - Kiểm tra đầu vào để ngăn chặn các cuộc tấn công tiêm nhiễm
  • Sử dụng Helmet - Một module giúp bảo mật ứng dụng Express bằng cách thiết lập các HTTP headers
  • Implement Rate Limiting - Giới hạn số lượng yêu cầu để ngăn chặn các cuộc tấn công brute force và DDoS
  • Keep Dependencies Updated - Cập nhật các gói để tránh các lỗ hổng bảo mật đã biết
// Sử dụng Helmet để bảo mật Express app
const express = require('express');
const helmet = require('helmet');
const app = express();

app.use(helmet());

// Implement rate limiting
const rateLimit = require('express-rate-limit');

const apiLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 phút
  max: 100, // giới hạn mỗi IP đến 100 requests mỗi windowMs
  message: 'Quá nhiều yêu cầu từ IP này, vui lòng thử lại sau 15 phút'
});

// Áp dụng cho tất cả các routes API
app.use('/api/', apiLimiter);

// Hoặc cho một route cụ thể
app.use('/api/login', loginLimiter);
